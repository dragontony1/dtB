수능이 끝나자마자 시작한 블로그 디자인 뜯어고치기 프로젝트가 약 1달의 대장정 끝에 완성되었다.

JavaScript 등 다양한 언어들을 사용하지는 않았지만, 대신 HTML과 CSS를 최대한으로 활용하였다. 이 과정에서 HTML과 CSS에는 정말 빠삭해져서 뿌듯하다. StyleSheet 2.1의 자세한 업데이트 내용은 Programming Diaries를 참고해주길 바란다.

일단 수능이 끝나고 나서부터 대부분의 시간을 블로그 코딩에 전념했다. 외부로부터 가져온 코드는 게시물마다 달려있는 페이스북 라이크 버튼 플러그인 뿐이고 나머지 코드는 정말 손수 다 작성한 것이다. 크롬에서는 오른쪽 마우스 클릭 > '페이지 소스 보기'를 클릭, 애플 사파리에서는 상단 메뉴바 > '개발자용' > '페이지 소스 보기' > '소스 코드'를 클릭하면 필자가 직접 작성한 코드들을 볼 수 있다. 여기를 클릭하면 이 블로그 전체의 CSS 코드를 볼 수 있다. CSS 코드는 전체가 600줄 가까이 되지만 반은 모바일 viewport용 CSS라 사실 그렇게 긴 코드라고 할 수는 없다.

무엇보다 가장 큰 변화는 단연 Jekyll을 사용하고 있다는 점일 것이다. Jekyll 서비스의 깃허브 README 파일에 의하면 Jekyll이란...

Jekyll is a simple, blog-aware, static site generator perfect for personal, project, or organization sites. Think of it like a file-based CMS, without all the complexity. Jekyll takes your content, renders Markdown and Liquid templates, and spits out a complete, static website ready to be served by Apache, Nginx or another web server. Jekyll is the engine behind GitHub Pages, which you can use to host sites right from your GitHub repositories.

이라고 한다. Jekyll을 어떻게 이 블로그에 적용시켰는지는 마찬가지로 Programming Diaries 카테고리에 글로 실어놓겠다.

오늘부로 다시 주기적으로 글을 업로드하도록 하겠다. 많은 기대 부탁한다.

아, 참고로 옛 블로그를 아카이브 보존용으로 다시 복원해놓을 계획이다.
