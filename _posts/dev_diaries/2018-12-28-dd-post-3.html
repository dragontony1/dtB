일단 전체적인 바 디자인은 완성했다. 하지만 효과까지 다 넣으려면 한참 남았다... 언제 저거 다 포지셔닝하냐;;;

이제 absolute 포지셔닝을 어떻게 쓰는건지 좀 알겠다! 자식을 상속하는 부모 기준으로 자식을 특정 방식으로/특정 위치에 배치하고 싶을 때 그냥 쓰면 오케이다. absolute 포지셔닝 설명만 듣고서는 잘 몰랐는데, 직접 사용해보니 어떻게 쓰는 건지 완전히 알겠다. 근데 vertical-align이 블럭 엘리먼츠에는 적용이 안 된다는건 좀 놀랐다.

nav 바 만드는 중; 생각보다 수월히 끝났다. 이제 hover selector만 더해주고 손만 좀 보면 끝이다. 글꼴 적용도 고민해야 하는데, 아직 할게 너무나도 많다. 근데 무엇보다 지금 망했다고 생각되는 것은 hover 적용하는 것이 생각보다 간단하지가 않다는 것이다...

당초 계획했던 효과는 .navIcon 클래스에 hover selector가 작동하면 .navLink가 display: none에서 block으로 바뀌는 것이다. 그런데, .navIcon이 .navLink의 부모 클래스가 아니라 안 된다... 그럼 temp 클래스 div로 두 개를 다 감싸서 해야 하나... 아 머리 터질 것 같다.
망했다; temp도 안 된다. 잘 생각해보면 될 리가 없다. 그럼 어쩔 수 없이 .navIcon 클래스 이름 바꿔서 .navLink의 부모 클래스로 다시 정의하는 수밖에... 그럼 밑에 css도 다 바꿔야하는데 어쩌지. 아악

겨우 됐다. 잠깐 설명을 하고 넘어가자면 앞쪽 코드에서 ul을 감싸던 div 태그를 없애고 ul 영역과 img랑 바로 맞닿게 한 다음에, ul display를 none으로 해놓고 .navIcon: hover일 때와 ul:hover일 때에만 ul display가 block이 되게 했다! 약간 임시방편 같지만 제대로 작동하기만 되면 상관없지 않은가! 대신 ul랑 img 맞닿게 하려고 img 주변에 20픽셀 씩 margin 줬던 거 padding으로 바꿔줬다.

드디어 당초 계획했던 기능 완성! 이제 디자인적 요소를 손 볼 차레다. 애니메이션은 나중에 추가하는 걸로.


엇 잠시만 지금 큰일났다.

navBar에 position: fixed를 걸어줬는데 갑자기 위치가 이상하게 됐다. 뭐가 문제지? 분명히 ul li a 모두 inline-box 잘 정의되어 있는데...

아니 잠시만...

An inline element whose position is set to absolute or fixed, obtains the capabilities of the block-level elements.
그럼 inline element를 fix 못 한다는 이야기인가? 도대체 왜?

어쩔 수 없이 ul에 fixed 적용해서 땜빵식으로 오류를 메꿨다.


걍 리셋해야겠다.. 땜빵에 뺌빵이 계속되니 너무 코드가 더러워진다. 일단 지금까지 해놓은 CSS는 그래도 좀 아까우니 주석 처리 해놓고 천천히, 처음부터 다시 하겠다! 그리고 nav 자손으로 div 하나가 전체를 감싸고 있었는데, 하다 보니 있어야 할 이유를 잘 모르겠다. 지우고 nav 자손으로 img와 ul만 있는 간단한 구조로 다시 해야겠다. 네비게이션 바 하나 디자인하는게 이렇게 어려워서야...
